# Прогнозирование времени измерений и уровня глюкозы с помощью RMTPP

## О проекте

Данный проект посвящён применению **повторяющихся отмеченных временных точечных процессов (Recurrent Marked Temporal Point Processes, RMTPP)** для прогнозирования событий в медицинских данных. Основная задача — предсказывать, **когда пациент с диабетом 1 типа сделает следующее измерение уровня глюкозы** и **каким будет это значение**.

### Мотивация

Сахарный диабет 1 типа (СД1) — хроническое заболевание, требующее постоянного мониторинга уровня глюкозы и точного расчёта доз инсулина. Ключевые проблемы существующих подходов:

- **Неравномерность измерений** — пациенты измеряют уровень глюкозы не через равные промежутки, а в зависимости от самочувствия, приёма пищи, физической активности и предыдущих показателей.
- **Сложность учёта всех факторов** — многие модели требуют данных о физической активности, питании и т.д., которые в реальной жизни трудно получить в полном объёме.
- **Риск ошибок** — около 64% пациентов допускают ошибки в расчёте доз инсулина, что приводит к гипо- и гипергликемии.

**Наш подход** использует только историю измерений (время, уровень глюкозы, дозу инсулина, количество углеводов) и не требует дополнительных труднодоступных данных.

### Ключевые отличия нашей модели

| Особенность | Традиционные модели | Наша RMTPP |
|-------------|---------------------|-------------|
| Данные | Требуют много дополнительных параметров (активность, сон, стресс) | Только время, глюкоза, инсулин, углеводы |
| Прогноз времени | Не предсказывают, когда будет следующее измерение | Предсказывают время следующего измерения |
| Учёт истории | Часто используют только текущее значение | Полная история событий через LSTM |
| Индивидуализация | Статические правила | Адаптация к паттернам пациента |

## Методология

### Что такое RMTPP?

RMTPP (Recurrent Marked Temporal Point Process) — класс моделей для анализа последовательностей событий, где каждое событие имеет:

- **Временную метку** \( t_i \) — когда произошло измерение
- **Маркер** \( y_i \) — показания сахара в крови (в ммоль/л) + дополнительная информация

В нашем контексте **событие измерения** включает:
- Факт измерения уровня глюкозы
- Полученное значение (в ммоль/л)
- Введённый инсулин (в единицах)
- Потреблённые углеводы (в граммах)


### Реализованная архитектура (PyTorch)

```python
class RMTPP(nn.Module):
    def __init__(self, input_dim, hidden_dim):
        super(RMTPP, self).__init__()
        self.lstm = nn.LSTM(input_dim, hidden_dim, batch_first=True)
        self.dropout = nn.Dropout(0.1)
        self.linear1 = nn.Linear(hidden_dim, hidden_dim//3)
        self.relu = nn.ReLU()
        self.linear2 = nn.Linear(hidden_dim//3, 4)  # 4 выхода: время + 3 параметра
```

### Функция потерь

Используется комбинированная MSE-потеря для двух целевых переменных:

```python
def masked_mse_loss(predictions, targets):
    loss_time = mse(predictions[:, 0], targets[:, 0])   # время следующего измерения
    loss_glucose = mse(predictions[:, 1], targets[:, 1]) # уровень глюкозы
    return (loss_time + loss_glucose) / 2
```

## Сравниваемые модели

Для оценки качества RMTPP сравнивались с:

| Модель | Описание |
|--------|----------|
| **RMTPP** | Наша рекуррентная модель с LSTM |
| **Linear** | Полносвязная нейронная сеть (базлайн) |
| **Poisson process** | Процесс с постоянной интенсивностью \(\lambda(t) = \lambda_0\) |
| **Hawkes process** | Само-возбуждающийся процесс \(\lambda(t) = \mu + \alpha \sum e^{-\beta(t-t_i)}\) |

**RMTPP более чем в 2 раза точнее** предсказывает время следующего измерения по сравнению с классическими точечными процессами.
